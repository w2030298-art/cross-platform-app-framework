# 监控告警配置

## Prometheus 配置要点

### 采集指标

| 服务 | 采集端点 | 间隔 | 关键指标 |
|------|---------|------|---------|
| API Gateway | :8080/metrics | 15s | 请求量、响应时间、错误率 |
| User Service | :8081/metrics | 15s | 注册量、登录量、DAU |
| Order Service | :8082/metrics | 15s | 订单量、GMV、转化率 |
| Content Service | :8083/metrics | 15s | 笔记发布量、互动量 |
| AI Service | :8087/metrics | 15s | 调用量、延迟、token消耗 |

### 告警规则

| 告警名称 | 条件 | 级别 | 通知方式 |
|---------|------|------|---------|
| API错误率过高 | 5xx比例>5%持续2分钟 | P1 | 飞书群+短信 |
| 响应时间过长 | P99>3s持续5分钟 | P2 | 飞书群 |
| 支付成功率下降 | 成功率<99%持续5分钟 | P0 | 飞书群+短信+电话 |
| 数据库连接池耗尽 | 可用连接<5 | P1 | 飞书群+短信 |
| 小程序审核被拒 | 事件触发 | P3 | 飞书群 |
| 内存使用过高 | >85%持续10分钟 | P2 | 飞书群 |
| 磁盘空间不足 | >90% | P1 | 飞书群 |
| 定时任务失败 | 触发后无完成事件 | P1 | 飞书群 |

## Grafana 仪表盘

### 业务看板
- 实时GMV / 订单量 / DAU / 转化率
- 各平台（微信/小红书/飞书）流量分布
- 会员数 / 新增 / 流失趋势
- 营销活动效果对比

### 技术看板
- API响应时间（P50/P90/P99）
- 错误率分布（按服务/接口）
- Pod资源使用率（CPU/内存）
- 数据库连接池/QPS/慢查询
- Redis命中率/内存使用
- 消息队列积压情况

### AI服务看板
- LLM调用量（按provider/model）
- AI响应时间分布
- Token消耗趋势
- 缓存命中率
- 错误率与降级统计
