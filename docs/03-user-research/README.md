# 3. 用户画像矩阵

## 3.1 微信小程序用户画像

### 画像1：社区宝妈

| 维度 | 描述 |
|------|------|
| 人口统计 | 25-35岁女性，二三线城市，家庭月收入8-15K |
| 行为特征 | 群聊活跃，爱拼团，关注性价比，平均每天使用微信4小时+ |
| 核心场景 | 亲子教育、家庭消费、社区团购、便捷服务 |
| 真实痛点 | 信息过载选择困难、信任成本高、群消息容易遗漏优惠 |
| 决策因子 | 朋友推荐>价格>品牌>颜值 |
| 可触达渠道 | 微信群、朋友圈广告、附近小程序 |

### 画像2：银发族

| 维度 | 描述 |
|------|------|
| 人口统计 | 55-70岁，刚学会微信，退休/半退休 |
| 行为特征 | 扫码为主，功能简单优先，信任实体店，怕被骗 |
| 核心场景 | 医疗挂号、生活缴费、社区服务、同龄社交 |
| 真实痛点 | 字体太小看不清、操作复杂不会用、怕点错扣钱、易被骗 |
| 决策因子 | 信任度>简单>功能>价格 |
| 可触达渠道 | 社区活动、子女推荐、线下扫码 |

### 画像3：都市白领

| 维度 | 描述 |
|------|------|
| 人口统计 | 25-40岁，一二线城市，月收入15K+ |
| 行为特征 | 通勤碎片时间，追求效率，品牌意识强 |
| 核心场景 | 外卖点餐、打车出行、办公工具、健身预约 |
| 真实痛点 | 同质化小程序太多没有真正节省时间、会员卡散落各处 |
| 决策因子 | 效率>体验>个性化>价格 |
| 可触达渠道 | 朋友推荐、公众号、搜索 |

### 画像4：个体商户

| 维度 | 描述 |
|------|------|
| 人口统计 | 30-50岁，各线城市，每天在店里 |
| 行为特征 | 手机管理生意为主，不懂技术，预算有限 |
| 核心场景 | 客户管理、营销工具、财务记账 |
| 真实痛点 | 不懂技术被坑过、预算有限选不好、运营不会做 |
| 决策因子 | 成本>效果承诺>操作简单>售后支持 |
| 可触达渠道 | 同行推荐、行业社群、地推 |

### 画像5：Z世代学生

| 维度 | 描述 |
|------|------|
| 人口统计 | 18-24岁，高校在读 |
| 行为特征 | 社交分享意愿强，追求新奇，组队习惯 |
| 核心场景 | 学习工具、社交娱乐、省钱攻略 |
| 真实痛点 | 粘性低快速流失、付费意愿弱、功能新鲜感消退快 |
| 决策因子 | 趣味>社交属性>免费>品牌 |

## 3.2 小红书用户画像

### 画像1：种草型消费者

| 维度 | 描述 |
|------|------|
| 人口统计 | 20-35岁女性，一二线城市，月消费3K-10K |
| 行为特征 | 先搜后买，信任真人测评，喜欢收藏 |
| 核心需求 | 真实产品体验、避坑指南 |
| 真实痛点 | 虚假种草多难辨别、看完记不住买什么 |

### 画像2：新锐品牌运营

| 维度 | 描述 |
|------|------|
| 人口统计 | 25-35岁，品牌方市场部 |
| 行为特征 | 每天看数据、焦虑流量、频繁调整策略 |
| 核心需求 | 爆款方法论、投放ROI最大化 |
| 真实痛点 | 算法捉摸不定、投流成本高、转化数据断层 |

### 画像3：KOL/KOC

| 维度 | 描述 |
|------|------|
| 人口统计 | 20-30岁，内容创作者 |
| 行为特征 | 日更笔记、焦虑涨粉、研究算法 |
| 核心需求 | 内容灵感、数据反馈、变现渠道 |
| 真实痛点 | 创作效率低、变现不稳定、粉丝增长遇瓶颈 |

### 画像4：MCN运营

| 维度 | 描述 |
|------|------|
| 人口统计 | 25-35岁，团队管理 |
| 行为特征 | 管理多个账号、批量操作、关注投入产出 |
| 核心需求 | 多账号管理、数据分析批量化 |
| 真实痛点 | 工具分散、重复劳动多、风控风险高 |

### 画像5：实体店主

| 维度 | 描述 |
|------|------|
| 人口统计 | 28-45岁，线下经营（餐饮/美业/零售） |
| 行为特征 | 想做线上但不会，依赖试错 |
| 核心需求 | 引流到店、本地获客 |
| 真实痛点 | 不会做内容、线上线下一体化差、广告不会投 |

## 3.3 飞书用户画像

### 画像1：企业IT负责人

| 维度 | 描述 |
|------|------|
| 人口统计 | 30-45岁，CIO/CTO |
| 行为特征 | 预算有限但需求多、排期长、对安全性要求高 |
| 核心需求 | 快速搭建业务系统、减少外包依赖 |
| 真实痛点 | 开发资源不足、需求积压严重、系统维护成本高 |

### 画像2：业务线主管

| 维度 | 描述 |
|------|------|
| 人口统计 | 30-45岁，部门总监 |
| 行为特征 | 管人又管事、天天开会、数据驱动意识萌芽 |
| 核心需求 | 流程自动化、数据可视化 |
| 真实痛点 | 信息孤岛、数据分散、跨部门协调效率低 |

### 画像3：一线操作工

| 维度 | 描述 |
|------|------|
| 人口统计 | 25-50岁，工厂/门店员工 |
| 行为特征 | 流转单据、汇报异常、学习新系统困难 |
| 核心需求 | 操作简单、减少重复录入 |
| 真实痛点 | 系统太多太复杂、学习成本高、领导催的数据不会整理 |

### 画像4：HR/行政

| 维度 | 描述 |
|------|------|
| 人口统计 | 25-35岁 |
| 行为特征 | 大量事务性工作、跨部门协调 |
| 核心需求 | 自动化审批、入职流程、考勤管理 |
| 真实痛点 | 重复性高易出错、跨部门协调难、手动统计耗时 |

### 画像5：项目经理

| 维度 | 描述 |
|------|------|
| 人口统计 | 28-40岁，PM |
| 行为特征 | 管进度/管资源/管风险、信息过载 |
| 核心需求 | 进度可追踪、异常可预警、文档可聚合 |
| 真实痛点 | 工具割裂（Project+飞书+Excel）、信息同步滞后 |
