# 9. 合规与风控

## 9.1 各平台合规要求

| 合规维度 | 微信 | 小红书 | 飞书 |
|---------|------|--------|------|
| **用户隐私** | 隐私协议强制弹窗，收集信息需授权 | 内容平台严格要求，用户数据不可外泄 | 企业数据安全，员工隐私保护 |
| **数据安全** | 不得存储wx.key，不得将数据转交第三方 | GMV/用户数据不可公开 | 企业级数据主权，ISO27001 |
| **内容审核** | 不得传播违法违规内容 | 种草内容需真实，禁止虚假宣传 | 企业内部内容免责 |
| **支付合规** | 微信支付需商户号，虚拟商品iOS限制多 | 平台内交易需开小红书店铺 | 企业支付/发票合规 |
| **资质要求** | 医疗/教育/金融需特殊资质 | 需品牌授权、广告资质 | 企业认证、安全评估 |
| **知识产权** | 不得侵犯商标/著作权 | 图片/视频版权、笔记原创性 | 企业内部知识资产管理 |

## 9.2 数据安全架构

```
┌──────────────────────────────────────────────────┐
│                   安全架构                         │
│                                                   │
│  传输层: TLS 1.3 + 证书固定                       │
│  认证层: OAuth2.0 / JWT + 双因子认证               │
│  授权层: RBAC + 数据行级权限                       │
│  加密层: AES-256 数据加密 / 密钥托管(KMS)         │
│  审计层: 全链路日志 / 操作审计 / 异常检测          │
│  合规层: GDPR / 等保2.0 / SOC2                    │
│                                                   │
│  特别要求:                                        │
│  - 微信: 不存储OpenID原始数据，脱敏后存储          │
│  - 小红书: 用户行为数据不出境                     │
│  - 飞书: 企业数据私有化部署选项                    │
└──────────────────────────────────────────────────┘
```

## 9.3 内容安全策略

### 微信端内容安全

- 接入微信内容安全API（msgSecCheck）对UGC内容实时检测
- 文本+图片双重审核机制
- 敏感词库实时更新（政治/色情/暴力/广告）
- 用户举报快速处理流程（24小时内响应）

### 小红书端内容安全

- 违规词检测与智能替换（接入小红书审核规范）
- 原创度检测（图片去重+文本查重）
- 虚假种草识别（过度美化+效果夸大标记）
- 广告合规标注自动提醒

### 飞书端数据安全

- 企业数据隔离（租户级别物理/逻辑隔离）
- 最小权限原则（仅申请必要API权限）
- 数据导出审计（谁在什么时候导出了什么数据）
- 合规认证（等保2.0 / ISO27001 / SOC2）

## 9.4 风险矩阵

| 风险 | 概率 | 影响 | 对冲策略 |
|------|------|------|---------|
| **平台政策变更** | 高 | 高 | 多平台分散风险；紧跟平台官方动态；保持框架灵活性 |
| **审核不通过** | 中 | 高 | 提前研究审核规则；避免灰色地带；预留1-2周审核时间 |
| **获客成本高于预期** | 高 | 中 | 先做免费版积累用户；内容营销获客；老带新裂变 |
| **同质化竞争** | 高 | 中 | 垂直行业深耕；AI差异化；数据壁垒构建 |
| **AI合规风险** | 中 | 中 | 使用合规API；内容审核双保险；区分辅助/决策责任 |
| **技术债务** | 中 | 中 | 约束30%时间还债；代码审查；自动化测试 |
| **商业化进度慢** | 中 | 高 | 3个月内验证付费意愿；设置关键指标红线 |
| **团队流失** | 中 | 高 | 核心成员股权激励；有意义的工作；清晰的成长路径 |
| **数据安全事件** | 低 | 极高 | 安全架构前置；渗透测试；数据加密；应急预案 |
| **小红书商业化收紧** | 中 | 高 | 内容工具而非违规工具；不触碰快速引流红线 |

## 9.5 应急预案

### 平台政策变更
- 24小时内完成影响评估
- 1周内调整产品策略
- 提前备好多端适配方案

### 审核被拒
- 记录拒审原因，建立知识库
- 建立审核前自查清单
- 关键时间节点提前2周提交审核

### 数据安全事件
- 1小时内启动应急响应
- 4小时内完成影响范围评估
- 24小时内完成修复和通报
- 7天内完成事后复盘和改进
