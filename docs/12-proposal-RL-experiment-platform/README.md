### 2.2 每个竞品的核心弱点

**TensorBoard**（最可能被你替代的目标）：
- UI停在2015年，深色主题刺眼，图表交互原始
- 没有团队协作，只有本地文件
- 没有实验对比的宏观视角，切换实验需要手动选目录
- 超参数只能看标量，缺乏结构化对比
- 不支持RL特有的：策略回放、Reward曲线对比、环境视频

**W&B**（最强大但也最贵）：
- 中国访问慢（需要海外服务器）
- Team版起价$50/人/月，中国实验室用不起
- 功能太臃肿，RL研究者80%功能用不到
- Sweep配置复杂，非RL研究人员很难上手

**SwanLab**（最直接的竞争者）：
- 2024年刚出来，功能仍在完善中
- 不支持华为昇腾以外的NPU
- 超参搜索、模型存储等高级功能尚缺
- 没有RL专项功能（策略回放、环境管理）
- 团队规模小，长期维护存疑

**MLflow**：
- 完全面向工程师而非研究者
- UI极简，没有美观的实验对比面板
- 需要自建服务器，对研究者不友好
- 缺少可视化的训练监控

### 3.3 RL专有的「杀手级功能」

这是区别于所有竞品的核心——**只有做过RL的人才会知道需要这些**：

#### 功能1：策略回放（Policy Replay）
```
传统方式：跑100万步训练，只看最终的Reward数字
ExpLab：训练过程中每N步自动保存策略快照
       → 在Dashboard中可选择任意步骤的策略
       → 在1x/2x/5x速度下回放Agent行为
       → 多策略并排对比回放
       → 一键导出GIF/视频用于论文/展示
```

#### 功能2：智能实验命名和分组
```
传统方式：exp_001, exp_002, exp_final_v3_fixed
ExpLab：自动结构化命名
       → 项目/算法/环境/实验组 四级分类
       → 自动提取关键超参生成实验名
       → 标签系统（baseline/ablation/hyperparam_sweep）
       → 智能搜索（"PPO LunarLander lr<0.001"）
```

#### 功能3：组会报告一键生成
```
传统方式：手动截TensorBoard图 → 粘PPT → 口头解释
ExpLab：点击「生成周报」
       → 自动汇总本周所有实验
       → 对比表（超参+指标+趋势图）
       → 最佳实验高亮
       → 生成PPT/LaTeX/Markdown格式
```

#### 功能4：Reward曲线的RL特有分析
```
传统方式：只画return vs steps的折线图
ExpLab：RL专项指标面板
       → 滑动平均 vs 原始值（多种窗口大小切换）
       → 多次seed的均值+方差带状图
       → 收敛速度自动检测（标注收敛点）
       → 探索率/策略熵变化趋势
       → 每个epoch的value/policy loss分离展示
```

| **实时通信** | WebSocket | 训练过程实时推送 |
| **部署** | K8s + 阿里云 | 国内节点，快速访问 |

### 4.3 Python SDK 设计

```python
import explab

# 初始化实验
exp = explab.init(
    project="PPO-LunarLander",
    group="hyperparam-sweep",        # 实验分组
    tags=["baseline", "ppo"],         # 标签
    config={
        "algorithm": "PPO",
        "env": "LunarLander-v2",
        "learning_rate": 3e-4,
        "gamma": 0.99,
        "clip_ratio": 0.2,
        "epochs_per_update": 10,
    }
)

# 训练循环中记录指标
for step in range(total_steps):
    reward = train_one_step()
    
    # 记录标量指标（自动附带step和时间戳）
    exp.log_metrics({
        "reward": reward,
        "policy_loss": policy_loss,
        "value_loss": value_loss,
        "entropy": entropy,
    })
    
    # 每N步保存策略快照（用于回放）
    if step % 10000 == 0:
        exp.log_policy(env, policy, step=step)
    
    # 每N步录制环境视频
    if step % 50000 == 0:
        frames = record_episode(env, policy, num_episodes=3)
        exp.log_video(frames, step=step, fps=30)

# 训练结束
exp.finish(status="completed", notes="收敛良好，最终平均Reward=260")
```
