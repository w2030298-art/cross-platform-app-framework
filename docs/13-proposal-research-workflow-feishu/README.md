# 方案2：飞书版科研工作流应用（研飞 ResearchFlow）

> 一站式科研团队协作：组会管理 + 实验记录 + 论文进度 + 导师查看

---

## 一、为什么选这个方向

### 1.1 你的不可替代优势

```
你独特的知识组合：
├── 完整的研究生经历 → 你亲身体验过科研全流程的混乱
├── 前端开发能力 → 你能在飞书生态里做出体验远超多维表格的产品
├── RL研究者视角 → 你知道实验记录、文献管理、组会报告的真实需求
│
飞书生态优势：
├── 飞书正在大力拓展高校市场（已有多个高校案例）
├── 飞书开放平台2500+API，开发成本低
├── 飞书应用商店分发，获客成本低
└── 多维表格+审批+机器人天然适合科研场景
```

### 1.2 市场验证

| 信号 | 数据 |
|------|------|
| 高校飞书用户 | 已有清华、北大、浙大等多所高校采购 |
| 现有竞品 | 衍因智研云定位"科研版飞书"，已获融资 |
| 科研管理痛点 | 80%+研究生使用Excel/Word记录实验，效率极低 |
| 飞书+科研案例 | 已有实验室在飞书上做OKR+文档协作+会议纪要 |
| 科研工具市场 | 全球科研信息化市场2025年预计超过200亿美元 |

### 1.3 为什么现在是窗口期

1. **飞书正在猛烈进攻高校**：免费版+高校采购，用户量快速增长
2. **衍因智研云验证了需求**：有人在做"科研版飞书"，但产品不成熟
3. **科研团队数字化痛点真实**：导师说"实验记录用Word"、组会PPT手动更新、文献用Zotero但无法团队共享
4. **飞书多维表格够强但不够专**：科研场景需要专门的字段类型和流程模板

### 2.2 衍因智研云（最直接竞品）详细对比

**衍因智研云做得好的**：
- 较早验证了科研团队数字化需求
- 有基本的实验记录、项目管理功能

- 没有AI辅助

## 三、产品定义

### 3.1 一句话定位

**研飞 — 让科研团队在飞书里完成所有科研协作**

### 3.2 核心功能分层

```
MVP（第1版，6周内交付）
├── 实验记录数字化
│   ├── 实验日志（富文本+图片+附件+代码片段）
│   ├── 实验模板（RL/通信/化学/生物等预置模板）
│   ├── 智能标签（自动识别实验类型/状态/结果）
│   └── 版本追踪（每次修改可追溯）
├── 组会管理
│   ├── 自动生成组会汇报（本周实验→PPT/Md文档）
│   ├── 组会日程+会议纪要（飞书日历+妙记集成）
│   ├── 导师红笔批注（在实验记录上直接批注）
│   └── 待办追踪（组会讨论出的Action Item自动变任务）
├── 文献与论文进度
│   ├── Zotero同步（导入文献库）
│   ├── 论文Gantt图（选题→实验→写作→投稿→审稿→接受）
│   ├── 论文进度周报（自动汇总各组员进展）
│   └── 导师一眼看板（所有学生的论文进度总览）
└── 飞书原生集成
    ├── 飞书消息推送（实验完成/异常/组会提醒）
    ├── 飞书多维表格存储（无缝对接现有数据）
    ├── 飞书审批（设备申请/出差审批/论文投稿审批）
    └── 飞书日历（实验预约/设备预约/组会排期）

增强版（3-6个月）
├── AI科研助手
│   ├── 实验总结AI（自动从实验记录生成摘要）
│   ├── 文献推荐AI（基于研究主题推荐相关论文）
│   ├── 论文写作AI（基于飞书文档的RAG问答）
│   └── 实验异常检测AI（数据异常自动告警）
├── 设备管理
│   ├── 设备预约日历（高价值设备排期）
│   ├── 设备使用记录（关联实验记录）
│   └── 设备维护提醒（自动到期提醒）
├── 多维表格深度集成
│   ├── 实验数据自动填表
│   ├── 多维表格→实验记录双向同步
│   └── 自定义仪表盘（导出看板到多维表格）
└── 课题组知识库
    ├── 实验方法沉淀（可复用的Protocol模板）
    ├── 组会纪要自动归类
    └── 新人Onboarding模板

生态版（6-12个月）
├── 实验数据管理（对接ExpLab方案1的实验可视化）
├── 实验伦理审批流程（IRB审批流）
├── 科研基金管理（多维表格+审批流）
├── 毕业论文进度管理（选题→开题→中期→答辩）
└── 跨课题组合作（飞书组织间协作）
```

### 3.3 核心场景流程

#### 场景1：研究生日常实验记录

```
传统方式：
  打开Word写实验记录 → 截图粘贴效果 → 保存到U盘 
  → 导师问起时翻半天找不到 → 版本混乱

研飞方式：
  飞书工作台打开【研飞】→ 选择实验日志模板
  → 富文本记录（插入代码/数据/图片）
  → 自动保存版本 → 导师可实时查看批注
  → 组会时一键生成汇报
```

#### 场景2：每周组会

```
传统方式：
  导师问"这周做了什么" → 学生翻Word找实验记录
  → 手动截图粘PPT → 导师口头提意见 → 无记录

研飞方式：
  组会前：研飞自动生成本周实验摘要
  → 导师在飞书文档中直接批注
  → 组会中：飞书会议+妙记自动纪要
  → 组会后：Action Item自动变任务卡+Duedate
  → 无人认领的任务自动提醒
```

#### 场景3：导师查看进度

```
传统方式：
  导师微信问"论文写到哪了" → 学生翻聊天记录找上次汇报
  → 导师自己记Excel管理学生进度 → 信息零散

研飞方式：
  导师打开【研飞看板】→ 一眼看到：
  → 各学生本周实验数量和状态
  → 各论文/项目的进度条和里程碑
  →逾期任务红标提醒
  → 需要批注的实验记录通知
```

### 4.2 飞书深度集成点

| 飞书能力 | 集成方式 | 研飞场景 |
|---------|---------|---------|
| **消息** | 机器人发送卡片消息 | 实验完成通知/组会提醒/导师批注通知 |
| **审批** | 自定义审批流 | 设备申请/出差审批/论文投稿审批 |
| **日历** | 创建日程+参会人 | 组会排期/设备预约/实验排期 |
| **多维表格** | 读写多维表格数据 | 实验数据结构化/进度跟踪/设备管理 |

| **妙记** | 获取妙记纪要 | 组会语音→文字→Action Item |
| **工作台** | 应用首页+小组件 | 实验进度概览/待办提醒/快捷入口 |
| **通讯录** | 获取组织架构 | 自动识别导师/学生关系 |
| **视频会议** | 会议+录制 | 在线组会+自动纪要 |

### 4.3 技术选型

| 层级 | 技术 | 理由 |
|------|------|------|
| **前端** | Vue3 + 飞书组件库 | 飞书官方推荐，开发效率最高 |
| **后端** | Java (Spring Boot) | 飞书SDK成熟，企业级稳定 |
| **数据库** | PostgreSQL | 主数据存储 |
| **搜索** | Elasticsearch | 实验记录/文献全文搜索 |

| **AI服务** | Python (FastAPI) | LLM API接入 |
| **部署** | 阿里云 K8s | 国内稳定 |

## 六、与方案1的协同效应

方案1（ExpLab）和方案2（研飞）不是互斥的，而是**天然互补**：

```
ExpLab（实验可视化）          研飞（科研协作）
    │                              │
    │  实验数据/指标/曲线 ──────→ │  实验记录/组会/论文
    │                              │
    │  ←──── 论文进度/导师反馈 ── │
    │                              │
    │         一体化画面：           │
    │  研究者在ExpLab跑实验        │
    │  → 数据自动同步到研飞         │
    │  → 组会时研飞自动生成汇报      │
    │  → 导师在研飞批注             │
    │  → 批注反馈回ExpLab           │
```

**协同变现路径**：
- 研飞免费引流 → ExpLab付费转化
- ExpLab数据反哺研飞 → 增强科研管理能力
- 两者打包销售 → 实验室版¥499/月
