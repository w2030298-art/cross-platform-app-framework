# 2. 三大平台生态深度拆解

## 2.1 微信小程序生态

### 2.1.1 核心能力矩阵

| 能力类别 | 具体API/能力 | 商业价值 |
|---------|------------|---------|
| **用户能力** | wx.login / 手机号快速验证 / getUserProfile | 身份识别→用户体系 |
| **支付能力** | 微信支付 / 商家转账 / 分账 | 交易闭环 |
| **分享传播** | onShareAppMessage / onShareTimeline / 朋友圈广告 | 社交裂变获客 |
| **消息触达** | 订阅消息 / 客服消息 / 模板消息 | 复购与留存 |
| **线下连接** | 扫一扫 / 附近的小程序 / NFC / iBeacon | O2O场景打通 |
| **直播能力** | 官方直播组件 / 小程序直播 | 实时互动变现 |
| **云开发** | 云函数 / 云数据库 / 云存储 | 低成本快速上线 |
| **搜索** | 页面收录 / sitemap / 搜索SEO | 被动获客 |
| **企业微信** | 企业微信客户联系 / 外部联系人 | B2C私域运营 |

### 2.1.2 审核与限制规则

- 包体限制：主包2MB，分包总计不超过20MB
- 审核周期：1-7个工作日（加急2-4天）
- 类目限制：部分行业需特殊资质（金融、医疗、教育等）
- 用户隐私：2023年起强制隐私协议弹窗，收集信息需用户授权
- 虚拟支付：iOS端禁止虚拟物品购买

### 2.1.3 流量入口全景（64个官方入口）

```
线下扫码 │ 附近的小程序 │ 公众号菜单 │ 公众号文章嵌入
聊天分享 │ 朋友圈分享 │ 搜索直达 │ 微信支付完成页
微信搜一搜 │ 小程序历史列表 │ 公众号关联推送 │ 企业微信
视频号直播 │ 视频号主页 │ 微信群聊 │ 微信广告拉动
微信红包 │ 微信卡包 │ 微信步数 │ 服务通知
```

### 2.1.4 真实开发者痛点（来自微信开放社区）

1. **性能类**：首屏加载超3秒用户流失40%，安卓端卡顿，cover-view组件异常
2. **开发类**：多端适配成本翻倍，API文档更新滞后于实际变化，开发工具不稳定
3. **审核类**：审核标准不透明，类目资质要求经常变动，被拒后无具体原因
4. **运营类**：无中心分发，冷启动流量几乎为零，订阅消息次数受限
5. **支付类**：iOS虚拟商品禁止购买，分账接口限制多，退款流程复杂

## 2.2 小红书小程序生态

### 2.2.1 核心能力矩阵

| 能力类别 | 具体能力 | 商业价值 |
|---------|---------|---------|
| **内容能力** | 笔记嵌入小程序卡片 / 话题关联 | 种草→拔草闭环 |
| **电商能力** | 小红书店铺 / 直播带货 / 商品笔记 | 站内交易闭环 |
| **搜索能力** | 搜索关键词 / 搜索联想 / 品类搜索 | 精准流量入口 |
| **用户互动** | 评论 / 私信 / 收藏 / 点赞 | 社区粘性 |
| **数据分析** | 创作者中心 / 品牌合作人数据 | 爆款洞察 |
| **广告体系** | 聚光平台 / 薯条 / 蒲公英 | 商业流量购买 |
| **直播能力** | 直播间 / 直播预约 / 直播回放 | 即时转化场景 |

### 2.2.2 内容-电商闭环模型

```
种草笔记 → 评论区互动 → 私信导流 → 店铺浏览 → 直播转化 → 复购沉淀
    ↑                                                          │
    └────── 用户UGC二次传播 ←──── 会员运营/社群维护 ←───────────┘
```

### 2.2.3 算法机制与流量逻辑

- **流量分发**：CES评分（点击率×互动率×收藏率×转发率）决定推荐量
- **长尾效应**：小红书笔记搜索流量可持续6-12个月（vs 视频7天衰减）
- **内容质量门槛**：原创>搬运，实拍>网图，专业>泛泛
- **违规高压线**：医疗声明、虚假种草、引流外链（2024年加强治理）

### 2.2.4 商家真实痛点（来自运营团队反馈）

1. **种草到购买断层**：用户在小红书种草，去淘宝下单，电商收入占比不足20%
2. **内容生产低效**：每篇笔记凭感觉发，不掌握爆款规律，MCN发布效率低
3. **投流成本攀升**：聚光平台CPC持续上涨，中小商家ROI倒挂
4. **算法变化快**：平台算法频繁更新，昨天有效的策略今天可能失效
5. **数据工具缺失**：官方数据仅限品牌合作人，第三方数据不够精准
6. **评论管理难**：大量评论需要逐条回复，差评处理不及时影响转化
7. **达人匹配低效**：靠人脉和经验选达人，缺乏数据驱动的匹配工具

## 2.3 飞书开放应用生态

### 2.3.1 核心能力矩阵

| 能力类别 | 具体API/能力 | 商业价值 |
|---------|------------|---------|
| **消息与群组** | 发送消息/卡片消息/群机器人/消息回调 | 实时通知与工作流触发 |
| **通讯录** | 部门/用户信息/组织架构 | 企业身份体系 |
| **日历** | 日程CRUD/空闲查询/会议室预定 | 会议协同 |
| **云文档** | 文档读写/多维表格/画板/幻灯片 | 内容协作与数据管理 |
| **审批** | 审批定义/审批实例/审批回调 | 业务流程自动化 |
| **视频会议** | 会议创建/录制/妙记/参会人管理 | 会议效率提升 |
| **工作台** | 应用首页/小组件/快捷入口 | 企业应用门户 |
| **AI能力** | 飞书Aily/智能伙伴/知识问答/多维表格AI | AI原生应用开发 |
| **低代码** | 飞书妙搭/多维表格插件/自动化工作流 | 业务人员自助开发 |
| **事件订阅** | 消息事件/审批事件/日历事件/文档事件 | 实时业务联动 |
| **身份认证** | SSO/免登/用户身份验证 | 企业安全集成 |

### 2.3.2 应用类型与分发

| 应用类型 | 触达方式 | 适用场景 |
|---------|---------|---------|
| **自建应用** | 企业内部工作台 | 内部系统、审批流程 |
| **应用商店应用** | 飞书应用目录公开上架 | 面向多企业SaaS |
| **网页应用(H5)** | 工作台内嵌 | 已有系统快速迁移 |
| **小程序** | 飞书客户端内运行 | 原生体验的轻量应用 |
| **机器人** | 群聊/单聊交互 | 自动化通知、工作流触发 |
| **小组件** | 云文档内嵌/工作台卡片 | 数据展示、快捷操作 |

### 2.3.3 飞书开放平台关键数据

- 2500+ 标准化服务端API
- 1000+ 行业头部企业客户
- 多维表格：200万+企业用户
- 低代码平台：数小时上线原型

### 2.3.4 企业真实痛点（来自客户案例与社区）

1. **系统孤岛严重**：ERP、MES、SAP、OA数据不互通，员工每天切换5-8个系统
2. **审批与执行割裂**：OA审批通过后，线下跟进无管理，缺少数据闭环
3. **无效会议多**：会议无纪要、无待办跟踪，无法评估会议质量（小鹏案例）
4. **IT开发资源不足**：业务部门需求排期6个月以上，无法快速响应
5. **供应链信息不同步**：供应商数据分散在多个系统，库存预警靠人工截图发Excel
6. **知识无法沉淀**：员工离职带走经验，新人培训周期长，重复犯错
7. **门店管理粗放**：巡检靠纸质表格，5分钟完成通报变为不可能
