# 1. 战略全景与市场定位

## 1.1 核心战略定位

```
┌─────────────────────────────────────────────────────────────┐
│                    三平台协同战略                              │
│                                                               │
│   微信小程序 ──→ 公私域流量闭环（触达+留存+变现）              │
│   小红书小程序 ──→ 种草-转化链路（内容+电商+品牌）            │
│   飞书开放应用 ──→ 企业效能提升（协同+自动化+数字化）           │
│                                                               │
│   底层共享：统一用户体系 / 数据中台 / AI能力层 / 组件库        │
└─────────────────────────────────────────────────────────────┘
```

## 1.2 战略选择矩阵

| 维度 | 微信 | 小红书 | 飞书 |
|------|------|--------|------|
| **流量类型** | 私域为主，公域为辅 | 公域为主，算法驱动 | 组织内部，封闭网络 |
| **用户心智** | 工具/服务消费 | 种草/生活方式发现 | 工作/效率提升 |
| **核心优势** | 14亿用户，支付闭环，社交裂变 | Z世代女性，内容带货，信任消费 | B端组织力，审批流程，数据孤岛打通 |
| **竞争壁垒** | 网络效应强，生态最成熟 | 内容护城河，社区文化 | 企业级安全性，深度业务场景 |
| **变现路径** | 电商/服务佣金，广告，SaaS | 电商交易抽佣，广告投放，品牌合作 | SaaS订阅，项目制，增值服务 |
| **进入时机** | 红海但细分仍有蓝海 | 快速增长期，商业化加速 | 深水区，需要行业深耕 |

## 1.3 不局限赛道的关键原则

- **不押注单一行业**：构建可配置的行业模板引擎，一套架构服务多个垂直场景
- **不锁定单一平台**：核心业务逻辑抽象，通过适配层对接不同平台API
- **不依赖单一变现**：SaaS+抽佣+增值服务+数据服务多元收入
- **不追求大而全首发**：每个平台选1-2个最小可行场景快速验证

## 1.4 三大平台的战略角色

| 平台 | 战略角色 | 核心指标 |
|------|---------|---------|
| 微信小程序 | 流量入口 + 变现场景 | DAU、GMV、付费转化率 |
| 小红书 | 品牌建设 + 种草引流 | 笔记互动率、搜索流量、粉丝增长 |
| 飞书应用 | 企业服务 + 深度绑定 | 企业数、-seat数、续费率 |

## 1.5 跨平台协同模型

```
小红书种草笔记 ──→ 微信小程序下单 ──→ 飞书客服群服务
       ↑                  │                    │
       │                  │                    │
  数据回流(复购)    用户画像同步          服务记录沉淀
       │                  │                    │
       └──────────── 数据中台统一 ←──────────┘
```

### 协同场景示例

**餐饮品牌全链路**
1. 小红书发布探店笔记 → 用户种草
2. 笔记中嵌入小程序卡片 → 用户进店
3. 微信小程序扫码点餐 + 会员注册 → 用户数据沉淀
4. 飞书门店群自动建群 → 客服跟进
5. 数据中台分析用户偏好 → 精准推送优惠券
6. 微信小程序推送复购优惠 → 用户二次到店
7. 小红书用户发布UGC → 新一轮种草
