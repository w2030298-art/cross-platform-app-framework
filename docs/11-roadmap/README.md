## 11.2 MoSCoW 优先级

### Must Have（P0，第1版必须）

- 微信小程序：扫码点餐/预约下单
- 微信小程序：会员注册+积分
- 微信小程序：基础营销（优惠券）
- 飞书应用：审批流程创建
- 飞书应用：消息通知
- 管理后台：商品/服务管理
- 管理后台：订单管理
- 统一登录（手机号）

### Should Have（P1，2个月内）

- 微信：裂变营销工具
- 微信：数据分析面板
- 飞书：自动化工单系统
- 飞书：多维表格集成
- 小红书：AI文案生成
- 跨平台：用户ID打通

### Could Have（P2，3-6个月内）

- 微信：直播组件
- 飞书：AI知识问答
- 小红书：竞品监控
- 数据中台：CDP
- AI引擎：智能推荐

### Won't Have（P3，6个月以后）

- 国际化多语言
- PaaS开放平台
- 第三方开发者市场
- 自有AI模型训练

## 11.3 关键指标红线（触发战略调整）

| 指标 | 红线 | 行动 |
|------|------|------|
| 种子客户转化率 | < 10% | 重新选择场景或调整定价 |
| 月流失率 | > 15% | 暂停增长投入，全力修复留存 |
| MVP上线延迟 | > 3个月 | 削减功能，回归最小核心 |
| 付费意愿调研 | < 30%愿意付费 | 重新选择方向 |
| 核心团队离职 | > 2人/季度 | 反思文化和管理问题 |
| MRR增长 | 连续2月负增长 | 停止扩张，聚焦产品-市场匹配 |

## 11.4 下一步行动

1. **本周**：选定1-2个最痛场景，确定优先做哪个平台的哪个方向
